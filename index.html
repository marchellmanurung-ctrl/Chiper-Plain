Rumus & Aturan Matematika

1. Proses Enkripsi (Ciphertext)

Setiap karakter Plaintext diubah ke nilai bilangan desimal ASCII-nya (P), kemudian digeser maju sebesar kunci (K) dengan dibatasi sisa pembagian 256:

C = (P + K) mod 256

2. Proses Dekripsi (Plaintext)

Untuk mengembalikan ciphertext (C) menjadi teks asli, nilai digeser mundur sejauh kunci (K). Diperlukan penambahan 256 sebelum modulo agar tidak terjadi nilai negatif dalam JavaScript:

P = (C - K + 256) mod 256
Mengapa Modulo 256? Standar pengkodean ASCII menggunakan representasi 8-bit (1 byte) yang menyimpan tepat 256 kombinasi angka desimal (0-255). Menggunakan modulo 256 memastikan tidak ada karakter hasil pergeseran yang keluar dari batas tabel ini.
